🔄 Understanding the Workflow

Development vs Production

Two Separate Environments:

  • Development: Your source code (src/ folder) - for testing
  • Production: Built code (dist/ folder) - for live website
  • Server: Only sees the dist folder
  • Changes: Only appear after rebuilding and uploading

Why This Process is Necessary

  1. React Build Process: Your source code must be compiled/optimized
  2. Performance: Production code is optimized for speed
  3. Security: Source code is not exposed on live server
  4. Reliability: Built code ensures consistent performance

⚡ Daily Update Process

Step 1: Make Your Changes

Edit your source files in the src/ folder:

  • Add new products in src/data/products.js
  • Update prices, descriptions, stock status
  • Change brand information in src/components/Header.jsx
  • Update home page content in src/pages/Home.jsx

Step 2: Test Locally

Always test before deploying:

  1. Run: npm run dev
  2. Open: http://localhost:5173
  3. Test all changes you made
  4. Check mobile view (browser dev tools)
  5. Verify images load correctly

Only proceed if everything works locally!

Step 3: Build Production Version

Create optimized code for live server:

  1. Stop development server (Ctrl+C)
  2. Run: npm run build
  3. Wait for "Build complete" message
  4. Check that dist folder was created/updated
Build output should show:
✓ Built successfully
✓ Production ready
✓ File size optimized

Step 4: Upload to Server

Replace the old dist folder on your server:

  1. Connect to your hosting (FTP, cPanel, etc.)
  2. Delete the old dist folder on server
  3. Upload the new dist folder from your computer
  4. Wait for upload to complete
  5. Verify all files uploaded successfully

Step 5: Verify Live Website

Test your live website:

  1. Open your live website URL
  2. Hard refresh (Ctrl+F5 or Cmd+Shift+R)
  3. Check that your changes appear
  4. Test new products, prices, etc.
  5. Test mobile version
  6. Check contact forms if updated

💡 Smart Workflow Tips

Batch Your Changes

Efficiency Strategy:

  • Don't upload after every small change
  • Make multiple changes then build once
  • Example: Add 5 products, update 10 prices, change stock - then upload once
  • Saves 50-70% time compared to individual uploads

🚨 Common Mistakes to Avoid

Critical Errors

❌ Making Changes but Not Uploading

Your local changes will NEVER appear on live site without rebuilding and uploading!

❌ Testing Only After Upload

Always test locally first. Broken code on live site loses customers!

❌ Not Keeping Backups

Keep previous working dist folder until new version is verified

❌ Uploading Partial Files

Always upload the complete dist folder. React needs all files together

🛠️ Troubleshooting Deployment Issues

Build Fails

  1. Check for syntax errors in your code
  2. Ensure all brackets and commas are correct
  3. Verify image URLs are accessible
  4. Check console for specific error messages

Upload Issues

  1. Verify internet connection
  2. Check server credentials
  3. Ensure sufficient disk space on server
  4. Try uploading smaller chunks if connection is slow

Changes Not Visible After Upload

  1. Hard refresh browser (Ctrl+F5)
  2. Clear browser cache
  3. Check if CDN is caching old version
  4. Verify correct dist folder was uploaded
