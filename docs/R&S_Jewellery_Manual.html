🚨 Golden Rule: Backup First, Change Second!

NEVER make changes to your website without creating a backup first!

⚡ Quick Backup Methods:

⚡ Quick Backup Methods

Method 1: Automated Backup Script (Easiest)

Double-click: BACKUP_SYSTEM.bat

This gives you options:

  • Quick Backup: Source code only (2 minutes)
  • Complete Backup: Everything including dist folder
  • Daily Backup: Automatic daily backups
  • Restore: Restore from any backup

Usage:

  1. Double-click BACKUP_SYSTEM.bat
  2. Choose option 1 (Quick Backup)
  3. Wait for "Backup Complete!" message
  4. Make your changes safely

Method 2: Git Version Control (Professional)

Industry-standard backup system:

See GIT_BACKUP_GUIDE.md for complete setup

Quick Start:

  1. Install Git from https://git-scm.com
  2. Open Command Prompt in project folder
  3. Run: git init (one time only)
  4. Before changes: git add . && git commit -m "Backup before changes"

Benefits:

  • Track every change with date/time
  • Restore any version instantly
  • Free cloud storage with GitHub
  • Professional version control

Method 3: Manual Copy (Simple)

Traditional folder copy:

  1. Right-click jewellery-store folder
  2. Choose "Copy"
  3. Paste in backup location
  4. Rename with date: jewellery-store-backup-2024-03-28

Pros:

  • No technical knowledge needed
  • Complete copy of everything
  • Easy to understand

Cons:

  • Uses more storage space
  • No version history
  • Manual process

🔄 Restoring from Backup

Using Backup Script

  1. Double-click BACKUP_SYSTEM.bat
  2. Choose option 4 (Restore from Backup)
  3. Select backup folder from list
  4. Confirm restore (Y/N)
  5. Files are restored instantly

Using Git

  1. Open Command Prompt in project folder
  2. See all backups: git log --oneline
  3. Copy commit ID of backup to restore
  4. Restore: git reset --hard COMMIT_ID
  5. Files are restored to that state

Manual Restore

  1. Delete current jewellery-store folder
  2. Copy backup folder to original location
  3. Rename back to jewellery-store
  4. Test website works

🚨 Emergency Scenarios

Computer Won't Start

  1. Get new computer or fix current one
  2. Install required software (Node.js, Git)
  3. Restore from cloud backup (GitHub/Google Drive)
  4. Or copy from external drive/USB
  5. Website is back online!

Website Completely Broken

  1. Don't panic! You have backups
  2. Identify when it worked last
  3. Restore from backup before the problem
  4. Test website works
  5. Figure out what caused the issue
  6. Make changes more carefully next time

Accidentally Deleted Important File

  1. Stop making any changes
  2. Restore from most recent backup
  3. File is back instantly
  4. Continue your work

⚡ Daily Operations

5-Minute Updates

💰 Change Price

  1. Open src/data/products.js
  2. Press Ctrl+F, type product name
  3. Find price: 2999
  4. Change the number
  5. Press Ctrl+S to save
  6. Refresh website - price updated!

📦 Update Stock Status

  1. Open src/data/products.js
  2. Find product
  3. Change inStock: true to inStock: false
  4. Save and refresh

🏷️ Add Sale Badge

  1. Open src/data/products.js
  2. Add originalPrice: 3999 (must be higher than price)
  3. Add badge: "SALE"
  4. Save and refresh

30-Minute Updates

📦 Add New Product

Pro Tip: Use PRODUCT_TEMPLATE.js as a starting point
  1. Open PRODUCT_TEMPLATE.js - copy the template
  2. Open src/data/products.js
  3. Paste template at the end (before the closing bracket)
  4. Fill in your product details:
    • id: Next number (check last product)
    • name: Product name
    • category: Must match category ID
    • price: Sale price in rupees
    • description: Product details
    • images: Image URLs
  5. Save and refresh - product appears!

🔄 Daily Deployment Workflow

Important: You MUST rebuild and upload after making any changes to see them on your live website!

Why You Need to Update Each Time:

  • Development: Your source code (src/ folder)
  • Production: Built code (dist/ folder)
  • Server: Only sees the dist folder
  • Changes: Only appear after rebuilding and uploading

Daily Update Process:

For Small Changes (5 minutes):
  1. Make your changes (add products, change prices, etc.)
  2. Build new version: npm run build
  3. Upload new dist folder to server
  4. Website updates instantly!

Practical Example - Tomorrow's Workflow:

Morning:
  • 9:00 AM - Add 3 new necklaces
  • 9:15 AM - Change prices on 10 items
  • 9:30 AM - Update stock status
Afternoon:
  • 2:00 PM - Build website (npm run build)
  • 2:05 PM - Upload dist folder
  • 2:10 PM - All changes live!

Smart Workflow Tips:

  • Batch Your Changes: Don't upload after every small change
  • Make Multiple Changes: Add 5 products, update prices, then build once
  • Saves Time: One upload for multiple changes
  • Update Schedule:
    • Daily: Price changes, stock updates
    • Weekly: New products, category updates
    • Monthly: Major content changes, brand updates

Common Questions & Answers:

Q: Do I need to upload everything each time?
A: Yes, but dist folder is optimized and small (usually 1-5MB)

Q: Can I update just one file?
A: No, React builds everything together. Must upload complete dist folder

Q: What if I make a mistake?
A: Keep previous dist folder as backup until new version works

🛠️ Troubleshooting

Common Issues & Solutions

💥 Website Not Loading

  1. Stop current server (Ctrl+C)
  2. Restart: npm run dev
  3. Check for error messages in terminal
  4. Clear browser cache (Ctrl+F5)

🖼️ Images Not Showing

  1. Test image URLs in browser
  2. Check for typos in URLs
  3. Ensure images are publicly accessible
  4. Check file paths for local images

📦 Products Not Displaying

  1. Check for syntax errors in products.js
  2. Ensure product IDs are unique
  3. Verify category names match exactly
  4. Check comma placement

🎨 Styling Issues

  1. Hard refresh browser (Ctrl+F5)
  2. Clear browser cache
  3. Restart development server
  4. Check for CSS syntax errors

Error Messages

Most Common: "Unexpected token" or "Missing comma"

Solution: Check for missing commas, brackets, or quotes in your code.
